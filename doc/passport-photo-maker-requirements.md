# Passport Photo Maker 需求文档

## 1. 项目概述

### 1.1 产品名称

Passport Photo Maker

### 1.2 产品定位

面向美国及英语市场用户的在线证件照工具站，帮助用户在浏览器内完成护照照片裁切、打印模板生成和规格检查，同时通过规格说明页、FAQ 页和教程页承接 SEO 流量，并接入 Adsense 获得广告收入。

### 1.3 产品目标

- 提供一个免登录、低门槛、浏览器本地处理的护照照片制作工具。
- 覆盖 `passport photo`、`2x2 photo`、`passport photo requirements` 等高意图关键词。
- 建立“工具页 + 规格内容页 + FAQ 页”的站点结构，提高广告展示机会。
- 用 MVP 验证 3 个关键指标：
  - 自然搜索流量是否持续增长
  - 工具页导出完成率是否达到可用水平
  - 内容页广告 RPM 是否具备放大价值

## 2. 用户与场景

### 2.1 目标用户

- 需要在线申请或更新美国护照的普通用户
- 需要准备数字照片或打印版证件照的出行用户
- 想在家完成低成本证件照制作的用户

### 2.2 核心痛点

- 不清楚护照照片尺寸和规格要求
- 不知道手机自拍是否能直接用于申请
- 线下拍照和打印成本偏高
- 现有免费工具广告过多、体验差、可信度低

### 2.3 核心使用场景

1. 用户搜索 `passport photo maker` 进入工具页
2. 上传本地照片
3. 按辅助框拖动和缩放头像
4. 导出数字版 JPG 或打印版模板
5. 如对规格不确定，再查看要求说明页和 FAQ

## 3. 产品范围

### 3.1 MVP 范围

- 上传 JPG / PNG 图片
- 2x2 inch 固定裁切框
- 图片平移与缩放
- 头部安全区域辅助框
- 导出数字版 JPG
- 导出 4x6 打印模板
- 美国护照照片要求说明页
- FAQ 页
- 隐私说明页

### 3.2 暂不纳入 MVP

- AI 自动抠图或自动换白底
- 多国护照/签证规格
- 账号系统和历史记录
- 云端图片存储
- 付费功能和支付流程

## 4. 信息架构

### 4.1 页面结构

- `/`
- `/passport-photo-maker`
- `/passport-photo-requirements-us`
- `/2x2-photo-size`

- `/digital-passport-photo-checker`
- `/faq`
- `/privacy`
- `/terms`

### 4.2 页面职责

- 首页：聚合工具入口、信任说明、核心 FAQ、内链到内容页
- 工具页：完成上传、裁切、导出
- 规格页：承接搜索流量并解释官方要求
- FAQ 页：覆盖尺寸、背景、表情、头部比例、打印方式等问题

## 5. 功能需求

### 5.1 图片上传

- 支持拖拽和点击上传
- 支持 JPG、JPEG、PNG
- 对超大图进行前端压缩和预览优化
- 上传后立即在浏览器内生成预览，不上传服务器

### 5.2 裁切编辑

- 裁切区域固定为 1:1
- 提供缩放滑块
- 支持拖动图片位置
- 显示头部建议区域辅助框
- 提示用户尽量保持纯白背景、自然表情、正面朝向

### 5.3 导出

- 导出数字版：适合在线申请使用
- 导出打印版：4x6 尺寸内排布多张 2x2 照片，便于冲印
- 导出前显示尺寸和使用提示

### 5.4 内容模块

- 规格说明：尺寸、背景、近照时间、眼镜、头部比例说明
- 常见失败原因：模糊、阴影、背景不纯、头部过大或过小
- 打印说明：如何在家打印或交给冲印店

## 6. SEO 与 Adsense 方案

### 6.1 SEO 策略

- 工具词：`passport photo maker`、`passport photo online`
- 规格词：`passport photo requirements us`、`2x2 photo size`
- 问题词：`can i take passport photo at home`、`passport photo background rules`

### 6.2 首批内容页策略

- 美国护照照片要求
- 2x2 尺寸详解
- 数字版照片要求
- 打印模板说明
- 护照照与签证照区别
- 手机拍摄注意事项

### 6.3 Adsense 布局

- 首页：首屏下方 1 个广告位，正文中部 1 个广告位
- 规格页：正文中部和结尾各 1 个广告位
- FAQ 页：问题列表中穿插 1 到 2 个广告位
- 工具结果页：仅在导出区域下方放 1 个广告位，避免干扰操作

## 7. 技术方案

### 7.1 技术栈建议

- 前端框架：Next.js 或 Vite + React
- 样式：Tailwind CSS
- 图片处理：Canvas API
- 导出：浏览器端 JPG / PNG 生成

### 7.2 架构原则

- 所有图片在本地浏览器处理
- 不依赖后端即可完成核心流程
- 内容页静态生成，利于 SEO
- 工具页优先保证移动端可用

### 7.3 核心实现点

- 使用 Canvas 生成裁切结果
- 用固定模板渲染 4x6 打印排版
- 使用本地状态存储用户编辑参数
- 对 object URL 做及时释放，避免内存泄漏

## 8. 合规与风险

### 8.1 合规要求

- 明确声明图片不会上传到服务器
- 不声称“官方认证通过”
- 规格说明页注明“请以官方最新要求为准”

### 8.2 主要风险

- 头部比例或背景检测不足，用户误以为一定通过
- 广告过多影响工具体验
- 多设备拍照质量差导致用户误判为工具问题

### 8.3 应对策略

- 工具输出明确使用提示和免责声明
- 优先将广告放在内容页而非核心编辑流程
- 后续再逐步增加更精细的规格校验

## 9. 核心指标

### 9.1 增长指标

- 自然搜索点击量
- 内容页收录数量
- 关键词进入前 20 的页面数

### 9.2 产品指标

- 上传成功率
- 导出完成率
- 页面停留时长
- 回访率

### 9.3 商业指标

- Adsense 页面 RPM
- 内容页广告 CTR
- 工具页与内容页的收益占比
