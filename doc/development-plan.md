# Passport Photo Maker 开发计划

## 1. 计划目标

在最短可控周期内完成 Passport Photo Maker 的 MVP，上线具备 SEO 能力的内容站和可用的本地证件照制作工具，并为后续放大做好结构准备。

## 2. 交付物

- 可运行的 Next.js 项目骨架
- 中英文双语路由和文案配置底座
- 首页、工具页、FAQ、隐私、条款页
- 4 个首批核心内容页
- 护照照编辑器 MVP
- 数字版导出与 4x6 打印模板导出
- 基础 SEO、分析埋点、广告位预留
- 测试用例与上线清单

## 3. 开发分期

### Phase 0：项目初始化

目标：搭建基础工程与规范，避免后续返工。

任务：

- 初始化 Next.js + TypeScript + Tailwind 项目
- 设计 locale 路由策略，首期支持 `zh` / `en`
- 建立 `app`、`components`、`features`、`lib` 目录结构
- 配置 ESLint、Prettier、基础测试框架
- 建立站点全局布局、页头、页脚、基础 SEO 配置
- 配置 sitemap、robots、404、基础元信息

产出：

- 可启动的项目骨架
- 基础布局组件
- 页面模板和内容配置结构

### Phase 1：内容站底座

目标：先让站点具备收录基础和内容承载能力。

任务：

- 完成首页
- 完成 FAQ、Privacy、Terms 页面
- 完成以下内容页：
  - `passport-photo-requirements-us`
  - `2x2-photo-size`
  - `passport-photo-print-template`
  - `digital-passport-photo-checker`
- 建立 CTA、相关推荐、FAQ 模块
- 预留广告位容器与埋点能力

产出：

- 一套可索引的内容站结构
- 首页到工具页和内容页的完整内链

### Phase 2：工具页 MVP

目标：打通上传、编辑、导出的核心闭环。

任务：

- 实现本地图片上传与格式校验
- 实现编辑器预览区
- 实现拖动与缩放能力
- 实现头部辅助框
- 实现数字版导出
- 实现 4x6 打印模板导出
- 增加错误提示、加载状态、免责声明

产出：

- `/passport-photo-maker` 可独立完成核心制作流程

### Phase 3：质量与上线准备

目标：保证可用性、可观测性和上线稳定性。

任务：

- 补充单元测试、组件测试、E2E 测试
- 做移动端适配与大图性能优化
- 接入分析埋点
- 接入 Adsense 容器
- 完成上线检查清单
- 进行生产环境冒烟测试

产出：

- 可上线 MVP
- 上线验收记录

## 4. 推荐执行顺序

1. 先搭站点框架与内容页，再做工具页
2. 工具页先完成预览与裁切，再完成导出
3. 广告与分析放在主链路可用之后接入
4. 测试优先覆盖导出算法和关键用户路径

## 5. 任务拆解清单

### 5.1 基础工程

- 创建项目与依赖
- 配置 Tailwind
- 配置路径别名
- 配置 lint / format / test
- 搭建全局布局和通用组件

### 5.2 内容系统

- 设计内容配置模型
- 编写 4 个核心内容页文案
- 编写 FAQ 数据
- 实现内容页模板
- 实现相关链接与 CTA 模块

### 5.3 编辑器系统

- 文件上传与校验
- 图片预加载与 object URL 管理
- 编辑器状态管理
- 预览区域与交互控制
- 头部辅助框叠层

### 5.4 导出系统

- 数字版裁切算法
- 4x6 打印模板布局算法
- 下载逻辑封装
- 导出异常处理

### 5.5 SEO 与商业化

- metadata 模板
- sitemap / robots
- FAQ 结构化数据
- 广告位组件
- 埋点事件定义

### 5.6 测试与上线

- 单元测试
- 组件测试
- Playwright E2E
- 移动端真机或模拟验证
- 上线前验收

## 6. 里程碑验收标准

### M1：内容底座完成

- 首页和核心内容页全部可访问
- metadata、内链、sitemap、robots 就绪
- 页面结构满足 SEO 基础要求

### M2：工具闭环完成

- 用户可上传照片并完成编辑
- 数字版和打印版导出均可成功
- 移动端关键链路可用

### M3：上线完成

- 测试通过
- 埋点和广告容器已接入
- 免责声明、隐私页、条款页齐全

## 7. 风险与排期建议

### 7.1 主要风险

- 没有现成代码，初始化阶段容易遗漏工程规范
- Canvas 导出在移动端可能出现性能和兼容性问题
- 内容页产出速度影响 SEO 启动节奏

### 7.2 建议节奏

- 第 1 周：完成 Phase 0 + Phase 1
- 第 2 周：完成 Phase 2
- 第 3 周：完成 Phase 3 并上线

如果是单人开发，这个节奏是偏紧但可执行的；前提是 MVP 严格控制在“美国护照照片 + 4 个核心内容页 + 无后端”这个范围内。

## 8. 上线后首轮迭代建议

- 增加拍摄指导模块
- 扩写更多 FAQ 与教程页
- 加入基础合规提示，不做自动通过承诺
- 扩展到 visa / ID photo 相关页面
